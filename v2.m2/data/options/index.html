Notes 1. Single-threaded downloads are handled by your browser's built-in download manager (multi-threading jobs are handled by "downloads.js" library and each segment is written into a temporary file. When all segments are fetched, the file is released to your browser's default download directory)
2. If "FileSystem" APIs are not yet supported by your browser, files will be downloaded by the browser's built-in download manager (at this moment Firefox does not support the "FileSystem" APIs. This extension will use "IndexedDB" APIs as the alternatives to the "FileSystem" APIs for small-sized files. Large-sized files are being downloaded by your browser's built-in download manager as the "IndexedDB" is not capable of allocating the disk)
